# Chorify 商业计划书 & 开发蓝图

> 一站式 **AI 视频营销平台** —— 对标筷子科技（Kuaizi.ai）的"编拍剪投管"核心链路。
> 本套文档既是 **商业计划书（BP）**，也是 **指导 `chorify3` 软件开发的产品 / 技术蓝图**。

---

## 这是什么

`chorify3` 要做的事，用一句话说：**让每一个商家都能像工厂流水线一样，工业化、规模化地生产并分发爆款营销视频。**

产品形态对标筷子科技，分为两大可见区块 + 一个隐性底座：

| 区块 | 模块 | 对应筷子 |
|---|---|---|
| **工作流** | 编导灵感 · 素材生产 · 合成量产 · 投放分发 | 编 / 拍 / 剪 / 投 |
| **服务** | 云盘 · 手机协同 · 直播切片 · 视频翻译 · 矩阵宝 · BGM市场 · 视频拆分 | 周边能力 |
| **底座** | 组织 / 成员 / 鉴权 / 资产 / 计费 | 管 |

## 一页速览

- **定位**：AI 视频营销 SaaS，服务中小商家、本地生活、跨境出海卖家与 MCN/代运营机构。
- **对标**：筷子科技（Kuaizi.ai），广州筷子信息科技有限公司，2015 年上线，"编拍剪投管"一体化 AIGC 平台，已服务全球 8500+ 企业、覆盖 800+ 城市、累计生成分发 8000 万+ 商业视频，2025 年 6 月完成近亿元 B 轮（Kuaizi 5.0），已启动数亿元 C 轮。
- **差异化**：轻量起步、价格亲民、Agent（小助手）驱动全流程、对中小商家与出海场景友好、开放 API 与数据自主。
- **阶段**：天使 / 种子轮 —— 先用一个"真骨架 + AI 能力可插拔"的 MVP 验证 PMF，AI 重活先 mock、后接真实供应商。
- **技术栈**：Next.js 16（前端 + 登录鉴权 + AI 代理）· Python FastAPI（AI 模块）· PostgreSQL 16 + Prisma 6 · Redis 7 · 阿里云 OSS（文件）· 阿里云短信（登录）。

> 详细融资与预算口径见 [01-商业计划书 · 第 9 节](./01-商业计划书.md#9-融资与预算天使--种子轮)。

## 命名

**Chorify** = *Chorus / Choreography* + *-ify*：把成百上千条素材"编排"成一场规模化的内容合奏。
品牌名只存在于一处：`apps/web/src/lib/brand.ts`（`NEXT_PUBLIC_APP_NAME`），改一次即可全局换名。
英文口号沿用核心动词：**Compose · Create · Cut · Cast · Manage**。
