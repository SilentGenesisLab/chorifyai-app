# Chorify 产品需求与功能规格（PRD）

> 给产品 / 设计 / 前后端工程师。逐模块说明"做成什么样、用到哪些数据与 AI 能力、mock 与 real 的边界、验收标准"。
> 路由与数据模型均**对齐当前 `chorify3` 仓库真实代码**（见 [03-技术架构与开发蓝图](./03-技术架构与开发蓝图.md)）。

---

## 0. 全局约定

### 0.1 信息架构 / 导航（已实现，见 `apps/web/src/lib/nav.ts`）

```
开始工作            /                首页
工作流
  编导灵感          /inspiration
  素材生产          /material
  合成量产          /compose
  投放分发          /distribute
服务
  云盘              /drive
  手机协同          /mobile
  直播切片          /live
  视频翻译          /translate
  矩阵宝            /matrix
  BGM市场           /bgm
  视频拆分          /split
```

### 0.2 通用规则

- **多租户**：一切业务数据归属 `Organization`；用户通过 `Membership` 加入组织并带 `Role`（OWNER/ADMIN/MEMBER）。所有列表/详情查询都必须带 `orgId` 过滤。
- **登录态**：未登录访问任意 `(app)` 页面 → 重定向 `/login`（已由 `proxy.ts` 实现）。
- **AI 长任务统一异步**：素材生成、混剪、翻译、切片等都走"**提交任务 → 轮询/订阅状态 → 产出落 OSS + 写库**"的 Job 模型，不阻塞请求。
- **Mock 优先**：当前所有 AI 重操作 `mock_mode=true`，返回占位结果；接真实供应商时只替换 AI 服务内部实现，前端契约不变。
- **空状态**：每个模块都要有"无数据"引导（CTA 指向创建/试用）。

### 0.3 用户故事书写格式

> 作为 `<角色>`，我希望 `<操作>`，以便 `<价值>`。 —— 后附**验收标准 AC**。

## 1. 首页 · 开始工作 `/`（已实现骨架）

**目标**：登录后第一屏，让用户"3 秒内知道能做什么、1 次点击进入工作"。

**内容区块**：
1. **顶部欢迎 + 组织名**（当前用户、所属组织，见 `constants.ts` 演示组织"尊爱日用 · 广州"）。
2. **AI 工具入口墙**（`HeroTools`）：编写脚本 / AI 影棚 / AI 复刻 / 元素替换 / 多语言配音 / 数字人视频 / 超级混剪 Pro… 横向卡片，点击进入对应能力。
3. **Agent 小助手入口**：对话式驱动（"帮我做 10 条护肤带货视频"），后续逐步打通到各模块。
4. **最近工程**（`RecentProjects` / `ProjectCard`，已从 DB 渲染）：卡片展示 `Project`（封面、名称、类型徽标"混剪/超级混剪/一键成片"、最后编辑时间）。

**用户故事**
- 作为商家，我希望进首页就看到最近做过的工程，以便接着上次继续。
  **AC**：按 `Project.updatedAt` 倒序展示本组织工程；点卡片进入对应工程；无工程时显示新建引导。
- 作为新用户，我希望首页有醒目的"新建/试用"入口，以便立刻开始。

## 2. 工作流模块

### 2.1 编导灵感 `/inspiration`（编）

**目标**：解决"不知道拍什么"。看什么火、学爆款结构、生成选题与脚本。

**功能**
- **灵感洞察**：按行业/平台/关键词浏览热门视频与趋势榜单。
- **爆款解构**：输入一条视频（链接/上传），AI 拆解其结构（开场钩子→卖点→促单→收尾）、脚本、节奏、BGM 风格。
- **AI 写脚本**：给定产品/卖点 → 生成分镜脚本（可一键流转到素材生产/合成量产）。
- **选题收藏**：存入云盘或工程。

**AI 能力**：LLM（脚本/解构）、视频理解（结构拆解）、检索（趋势）。
**Mock→Real**：mock 返回示例榜单与示例脚本；real 接 LLM + 视频解析供应商 + 平台数据源。
**数据**：选题/脚本可作为 `Asset`（type=MATERIAL）或附属于 `Project`。

**用户故事**
- 作为商家，我希望粘贴一条爆款链接就拿到它的脚本套路，以便照着做自己的。
  **AC**：提交后生成"钩子/卖点/CTA/时长/BGM 风格"结构化结果，可一键"用此脚本新建工程"。

---

### 2.2 素材生产 `/material`（拍）

**目标**：解决"没素材"。从 0 生成多模态素材原料。

**功能（对应筷子的拍）**
| 子能力 | 输入 | 输出 | 后端 type |
|---|---|---|---|
| AI 影棚 | 商品图 + 场景描述 | 商品场景图/视频 | `ai_studio` |
| AI 复刻 | 参考视频 | 同结构新视频 | `replica` |
| 元素替换 | 视频 + 目标元素 | 替换后视频 | `element_swap` |
| 多语言 AI 配音 | 文本 + 音色 + 语种 | 配音音频 | `dubbing` |
| 数字人视频 | 脚本 + 数字人形象 | 口播视频 | `digital_human` |
| 文生图/图生图 | 提示词/图 | 图片素材 | （扩展）|

**接口**：已在 `apps/ai-service/app/routers/material.py` 定义
`POST /api/material/generate {type, prompt?, image_url?, options?}` → `Job{job_id,status,type,result_url?}`；
`GET /api/material/jobs/{job_id}` 轮询。
**Mock→Real**：mock 返回 `queued`/占位 URL；real 分发到 Seedance（文生视频）/ TTS / 数字人等供应商。
**数据**：产出写入 `Asset`（VIDEO/IMAGE/AUDIO），落 OSS 得 `url`/`thumbnailUrl`/`durationSec`。

**用户故事**
- 作为没有出镜意愿的商家，我希望用数字人 + 配音生成口播视频，以便不真人上镜也能出片。
  **AC**：选形象+音色+脚本→提交任务→任务完成后素材出现在云盘且可用于合成量产。

---

### 2.3 合成量产 `/compose`（剪）—— **核心变现模块**

**目标**：解决"产能"。把少量优质素材**批量裂变**成上百上千条可投放短视频。

**三种工程类型（已在数据模型 `ProjectType` 预留）**
| 类型 | 枚举 | 说明 |
|---|---|---|
| 智能混剪（经典版） | `SMART_MIX` | 按片段库 + 规则随机组合出 N 条 |
| 超级混剪 Pro | `SUPER_MIX` | 大模型理解语义的高质量混剪 |
| 一键成片 | `ONE_CLICK` | 给商品/脚本，端到端自动出片 |

**功能**
- 新建工程：选类型、设画幅（默认 1080×1920 竖屏，见 `Project.width/height`）。
- 配置素材：从云盘选片段/口播/BGM/字幕模板。
- 设定产出数量 N（`comboCount`），一键生成；任务进度可见。
- 产出预览、批量下载、推送到投放分发。

**接口**：已在 `apps/ai-service/app/routers/compose.py` 定义
`POST /api/compose/mix {project_id, count}` → `MixResponse{combos:[{combo_id,status,preview_url}]}`。
**Mock→Real**：mock 返回 N 个占位 combo；real 接 ffmpeg/渲染管线（拼接、转场、字幕、变速、去重）。
**数据**：`Project`（status: DRAFT→PROCESSING→READY，`comboCount`），产出 combo 作为 `Asset`（type=FINISHED）。

**用户故事**
- 作为带货商家，我希望一键生成 100 条不同组合的短视频，以便喂给投放算法测跑。
  **AC**：设定 N=100 提交后工程进入 PROCESSING；完成后 READY 且产出 100 条可预览/下载/转分发。

---

### 2.4 投放分发 `/distribute`（投）

**目标**：把成片**批量分发到多平台多账号**并回收数据。

**功能**
- 选成片（来自合成量产/云盘）+ 选目标账号矩阵（关联"矩阵宝"账号）。
- 设定平台、发布计划（定时/即时）、文案/话题。
- 批量发布进度跟踪（`publishDone/publishTotal`）。
- 数据回收：播放、点赞等（`views`/`likes`），统一看板。
- 智能互动（评论/私信自动回复）—— 进阶。

**支持平台**（已在 `Platform` 枚举）：TikTok / 抖音 / 快手 / YouTube。
**数据**：`Distribution`（title, platform, accountCount, videoCount, publishDone/Total, views, likes）。已有种子数据演示。
**Mock→Real**：mock 展示分发记录与假数据；real 接各平台开放接口/RPA 适配层（抽象 `DistributionAdapter`，单平台变动不影响主链路）。

**用户故事**
- 作为运营，我希望把 80 条视频分发到 6 个抖音号并看到每条的播放，以便筛出爆款。
  **AC**：批量发布后实时显示 done/total；列表展示每个分发任务的账号数/视频数/播放/点赞。

## 3. 服务模块

> 服务模块围绕工作流提供"输入端 / 资产端 / 增强端"能力。早期多为占位页（`ModulePlaceholder`），按路线图逐个落地。

### 3.1 云盘 `/drive`
**目标**：素材与成片的统一存储、分类、协同。
**功能**：文件夹树（`Folder` 自引用 parentId）、文件列表（`Asset`：视频/图片/音频/工程/成片/素材）、上传（走 OSS）、预览、搜索、移动/归档/删除（`AssetStatus`）。
**数据**：`Folder` + `Asset`（已有种子数据）。**AC**：能新建文件夹、上传文件落 OSS、按类型筛选、移动与软删除。

### 3.2 手机协同 `/mobile`
**目标**：用手机补齐"实拍素材"输入。
**功能**：Web 端展示二维码 → 手机扫码打开轻量上传页 → 拍摄/选择素材直传 OSS → 自动归入当前组织云盘。
**AC**：扫码页可上传，文件秒级出现在云盘。**Mock→Real**：mock 跳过真实直传。

### 3.3 直播切片 `/live`
**目标**：把长直播录像自动切成可投放的爆款片段。
**功能**：上传/拉取直播流 → 大模型理解内容 → 自动定位高光 → 切片 + 字幕 → 落云盘。
**AI 能力**：ASR + 视频理解 + 剪辑。**AC**：上传一场直播，产出 N 个带字幕高光片段。

### 3.4 视频翻译 `/translate`
**目标**：一套素材出海多语言本地化。
**功能**：上传视频 → ASR 转写 → 翻译 → 多语言字幕 + 多语言配音（可声音克隆）→ 合成新视频。
**AC**：选目标语种，产出带目标语字幕/配音的视频。**Mock→Real**：real 接 ASR + MT + TTS。

### 3.5 矩阵宝 `/matrix`
**目标**：多平台多账号的**账号资产管理**，是投放分发的账号来源。
**功能**：账号授权/绑定、分组、健康度/状态、与分发任务关联。
**AC**：能添加账号、分组、在分发时按矩阵选号。

### 3.6 BGM市场 `/bgm`
**目标**：提供可商用音乐库，供合成量产/素材生产选用。
**功能**：分类/情绪/时长检索、试听、收藏、授权信息标注。
**AC**：能搜索试听并在工程中选用某条 BGM。

### 3.7 视频拆分 `/split`
**目标**：把长视频拆成片段库，喂给智能混剪。
**功能**：上传长视频 → 按镜头/场景/字幕自动分段 → 批量入云盘片段库。
**AC**：上传一条长视频，产出可单独管理的片段集合。

## 4. 管理底座（管）

> 不在侧边栏独立成页，但贯穿全产品；早期最小实现。

- **组织与成员**：创建/切换组织，邀请成员，分配角色（OWNER/ADMIN/MEMBER），见 `Organization`/`Membership`/`Role`。
- **计费与配额**：`Plan`（FREE/PRO/ENTERPRISE）+ Credits 余额与消耗（路线图后期落地）。
- **数据看板**：跨模块的产量/分发/效果统计。
- **AC（早期）**：登录后自动归入演示组织；能看到组织名与成员；权限影响可见操作（后续细化）。

## 5. 登录 `/login`（已实现）

**目标**：手机号 + 短信验证码登录（无密码）。
**流程**：输入手机号 → `POST /api/auth/send-code`（dev 下 `SMS_PROVIDER=mock`，验证码打印到服务端控制台）→ 输入验证码 → `POST /api/auth/verify` → 签发 JWT 写入 httpOnly cookie → 跳回 `next` 或首页。
**AC**：错误码提示、验证码有效期与频率限制（Redis）、登录后自动加入演示组织、`/api/auth/me` 返回当前用户、`logout` 清除会话。

## 6. 跨模块的"一条主线"验收（North-Star 流程）

把上述模块串成一条端到端验收用例（也是 Demo 脚本）：

> 登录 → 编导灵感生成脚本 → 素材生产出口播/素材 → 合成量产批量出 50 条 → 投放分发到 3 个账号 → 回看播放数据 → 成片沉淀到云盘。

**这条主线全部跑通（即便 AI 仍为 mock）= MVP 成立。** 真实化只是把每段的 mock 换成供应商实现。
