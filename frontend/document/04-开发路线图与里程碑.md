# Chorify 开发路线图与里程碑

> 把商业目标（01）、产品规格（02）、技术蓝图（03）拧成一条**可执行的分迭代交付计划**。
> 核心原则：**先真骨架、AI 重活先 mock 后 real；每个迭代都要能完整演示（Demo-able）。**

---

## 0. 总览：六个迭代 → 三个商业里程碑

| 迭代 | 主题 | 一句话目标 | 商业里程碑 |
|---|---|---|---|
| **Iter 1** | 真骨架 ✅ | 登录 + 应用外壳 + 首页 + 数据模型 + AI 骨架 | M0–M3 起步 |
| **Iter 2** | 合成量产闭环 | 把"批量出片"做成真实可用（核心变现）| M0–M3 |
| **Iter 3** | 素材生产真实化 | 接通第一个真实 AI 供应商 + 编导灵感 | M3–M6 |
| **Iter 4** | 投放分发闭环 | 成片 → 多号分发 → 数据回收 | M3–M6 |
| **Iter 5** | 服务补齐 + 出海 | 翻译/直播切片/拆分/手机协同/BGM/矩阵宝 | M6–M12 |
| **Iter 6** | 商业化 | Credits 计费 + 套餐 + 数据看板 + 私有化 | M6–M12 |

> North-Star 主线（见 02 第 6 节）：**登录 → 灵感脚本 → 素材 → 批量成片 → 分发 → 数据 → 云盘沉淀**。
> 这条线在 Iter 1–4 逐段由 mock 转 real，**Iter 4 末整条主线真实跑通 = PMF 验证就绪**。

---

## Iter 1 · 真骨架 ✅（已落地）

**目标**：搭好"能登录、能进应用、能看到真实数据"的地基。

**已交付**
- [x] Monorepo：pnpm workspace + Turborepo + docker-compose（PG16 + Redis7）
- [x] 短信验证码登录 + JWT(jose) httpOnly cookie + `proxy.ts` 路由保护
- [x] `/api/auth/{send-code,verify,me,logout}`
- [x] 应用外壳：侧边栏（工作流 + 服务，`nav.ts`）+ `(app)/layout.tsx`
- [x] 首页 开始工作：AI 工具墙 + 最近工程（Server Component 直连 Prisma）
- [x] Prisma 数据模型 7 表 + 枚举 + 种子数据（对齐截图）
- [x] FastAPI 骨架：`/health` + `material` + `compose`（mock_mode）
- [x] OSS / SMS 的 mock + aliyun 双 provider 抽象

**验收**：手机号登录（验证码读控制台）→ 进首页看到种子工程 → 侧边栏可达 12 个模块页。

---

## Iter 2 · 合成量产闭环（核心变现，优先级最高）

**为什么先做它**：合成量产是商家最强付费意愿点（"一键出上百条"），也是 Credits 计费的主要消耗场景。即便素材仍靠上传，先把"批量产能"跑通最能验证价值。

**范围**
- [ ] 业务 CRUD（Next.js）：`/api/projects`、`/api/assets`、`/api/folders`（带 orgId 鉴权）
- [ ] 云盘 `/drive` 真实化：文件夹树 + 文件列表 + **OSS 真实直传**（`OSS_PROVIDER=aliyun`）
- [ ] 合成量产 `/compose`：新建工程（选类型/画幅）→ 从云盘选素材 → 设数量 N → 提交
- [ ] AI 服务 `compose/mix` 接**真实渲染管线**（ffmpeg：拼接/转场/字幕/变速/去重），产出落 OSS
- [ ] 统一 **Job 模型**落地（`Job` 表 + Redis 队列 + worker + 轮询）
- [ ] 工程状态机：DRAFT → PROCESSING → READY；产物写 `Asset(type=FINISHED)`

**交付/验收（Demo）**：上传几个片段 → 新建智能混剪工程 → 一键生成 20 条 → 进度可见 → 完成后预览/下载，成片落云盘。
**退出条件**：真实产出可投放的混剪视频；单条生产成本可度量。

---

## Iter 3 · 素材生产真实化（接第一个真实生成供应商）

**范围**
- [ ] 素材生产 `/material` 前端：五类能力的表单与产出页（AI 影棚/复刻/元素替换/配音/数字人）
- [ ] AI 服务 `material/generate`：接**首个真实供应商**（建议先做**多语言配音 TTS** 或**文生视频**，成本可控、见效快）
- [ ] 编导灵感 `/inspiration`：接 LLM 做"AI 写脚本 + 爆款解构"；产出可一键"用此脚本新建工程"
- [ ] 供应商 `Provider` 接口固化（mock/real 同接口），`mock_mode` 与配置切换
- [ ] 素材产出统一回流云盘，可直接喂给 Iter 2 的合成量产

**交付/验收**：输入卖点 → 生成脚本 → 生成配音/数字人口播 → 该素材进入云盘并用于一次真实混剪。
**退出条件**：素材生产 → 合成量产两段真实打通。

---

## Iter 4 · 投放分发闭环（打通 North-Star 主线）

**范围**
- [ ] 矩阵宝 `/matrix`（最小版）：`SocialAccount` 模型 + 账号绑定/分组（先支持 1–2 个平台）
- [ ] 投放分发 `/distribute`：选成片 + 选账号矩阵 + 发布计划 → 批量发布
- [ ] 分发适配层 `DistributionAdapter`：抽象各平台，单平台变动不影响主链路（先接 1 个真实平台，其余 mock）
- [ ] 数据回收：`publishDone/Total`、`views`/`likes` 回写 `Distribution`，统一看板
- [ ] AI 代理中接入**配额/Credits 预扣**（为 Iter 6 计费铺路）

**交付/验收（整条主线）**：登录 → 灵感脚本 → 素材 → 批量成片 → 分发到多个账号 → 看到回收数据 → 成片沉淀云盘。
**退出条件**：North-Star 主线**端到端真实跑通**（至少一个平台真实分发）= PMF 验证就绪，可启动种子客户与下一轮融资数据采集。

---

## Iter 5 · 服务补齐 + 出海

**范围**
- [ ] 视频翻译 `/translate`：ASR + 翻译 + 多语言字幕/配音（出海关键）
- [ ] 直播切片 `/live`：长直播 → 高光定位 → 切片 + 字幕
- [ ] 视频拆分 `/split`：长视频 → 片段库，反哺智能混剪
- [ ] 手机协同 `/mobile`：扫码手机直传素材入云盘
- [ ] BGM市场 `/bgm`：可商用音乐库检索/试听/选用
- [ ] 出海：海外平台（TikTok/YouTube）分发为一等公民

**交付/验收**：一套中文素材 → 翻译/配音成多语言 → 分发到海外平台；一场直播 → 自动切出多条高光。

---

## Iter 6 · 商业化

**范围**
- [ ] Credits 计费：`CreditLedger` 模型 + 充值 + 各 AI 操作按量扣减 + 余额/账单
- [ ] 套餐：`Plan`（FREE/PRO/ENTERPRISE）权益与配额、升级路径、水印策略
- [ ] 组织/成员管理 UI：邀请、角色权限细化（OWNER/ADMIN/MEMBER）
- [ ] 数据看板：跨模块产量/分发/效果统计，经营指标（转化/复购/留存）
- [ ] 私有化/数据导出（Enterprise 卖点）、SSO

**交付/验收**：新用户走免费额度 → 触达上限引导付费 → 充值 Credits → 按量消耗有账单；管理员可邀请成员并分配权限。

## 关键工程顺序（依赖关系）

```
Iter1 真骨架 ✅
  └─> Iter2 业务CRUD + OSS真实 + Job模型 + 渲染管线   （地基之上第一块承重墙）
        ├─> Iter3 素材生产/编导灵感（喂素材给混剪）
        └─> Iter4 投放分发 + 矩阵宝（消费成片）+ Credits预扣
              └─> Iter5 服务补齐 + 出海
                    └─> Iter6 商业化放量
```
> **Job 模型 + Provider 抽象**（Iter2/Iter3）是所有后续 AI 模块的公共底座，务必先做扎实，后面每个 AI 能力都是"加一个 provider + 一条路由"。

- [ ] 能在本地一条命令起服务并完整演示该迭代主路径

- [ ] README / 本路线图同步更新进度勾选

## 风险与节奏建议

- **不要过早自研模型**：Iter 2–5 全部用供应商 API，把钱花在跑通闭环与拿客户，而非重资产。
- **先深后广**：先把合成量产 + 一个真实供应商打透（Iter 2–3），再横向铺服务模块（Iter 5）。
- **成本随真实化上升**：从 Iter 2 起就埋 Credits 预扣与单条成本度量，避免规模化后毛利失控。
- **合规前置**：生成内容审核、平台分发规则适配在 Iter 4 同步建设，别等出事。
